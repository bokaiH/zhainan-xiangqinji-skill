# 02-content-patterns：内容模式提炼

## 核心内容形态

公开视频标题和公开摘要显示，其内容不是纯理论讲解，而是强案例驱动：用相亲、约会、聊天记录、人物现场表现来做复盘。

高频栏目感包括：

- “拍100个直男相亲”
- “拍100个直男约会”
- “宅男相亲记”
- “相亲分析”
- “聊天案例”
- “脱单攻略”

## 高频问题类型

### 1. 相亲后断联或冷淡

典型问题：

- 第一次相亲后女生不理。
- 女生说“没感觉”。
- 相亲多次仍是母单。
- 用户自认为聊得还行，但对方没有后续。

对应 Skill 模型：现场大于自述；反馈信号分级；条件不是吸引。

### 2. 男性现场表现问题

典型问题：

- 约会紧张不敢牵手。
- 气场不足、弱者感爆棚。
- 内心自卑，用装、吹、伪装来掩饰。
- 全程冷脸、带不动、不会制造轻松感。

对应 Skill 模型：需求感和弱者感；推进关系看窗口；平视框架。

### 3. 聊天问题

典型问题：

- 消耗型聊天。
- 目的性太强。
- 夺命连环问。
- 女生冷漠后如何判断是否还有希望。
- 只会认真、照顾、解释，却无法形成吸引。

对应 Skill 模型：讨好不是情绪价值；不拒绝不等于喜欢；先判断人再判断话术。

### 4. 关系推进问题

典型问题：

- 羞于和女生升高关系。
- 约会中如何自然升级。
- 恋爱小白被女生撩得手足无措。
- 有窗口不行动，或没窗口硬冲。

对应 Skill 模型：推进关系要看窗口，不是硬冲。

### 5. 价值观与防御力

典型问题：

- 付费关系风险。
- 只索取不投入的关系。
- 不值得继续投入的对象。
- 社交防御力不足。

对应 Skill 模型：渠道、防御力与现实边界。

## 隐含方法论

两个 skill 整合后，最稳定的方法论可以概括为：

1. 不只看“男生条件”，更看“互动现场”。
2. 不只给话术，更强调状态、节奏和框架。
3. 不只教追求，也教识别不值得投入的关系。
4. 不只讲恋爱技巧，还把外在展示、社交渠道、生活结构纳入分析。
5. 不凭用户自述下结论，而是把事实、解释和愿望拆开。

## 适合蒸馏成 Skill 的方向

最适合做成“相亲/约会案例复盘器”，帮助用户把一次失败或模糊互动拆成：

1. 事实还原。
2. 对方投入度判断。
3. 男方行为问题。
4. 情绪价值与聊天节奏。
5. 关系推进窗口。
6. 修正动作与禁区。
7. 边界风险与是否该撤。

## 最终整合出的 7 个模型

1. 现场大于自述，先观察再行动。
2. 平视框架，不要讨好附和。
3. 反馈信号分级，不拒绝不等于喜欢。
4. 条件不是吸引，现场状态才进入关系。
5. 需求感和弱者感会吞掉吸引力。
6. 推进关系要看窗口，不是硬冲。
7. 渠道、防御力与现实边界同样重要。
